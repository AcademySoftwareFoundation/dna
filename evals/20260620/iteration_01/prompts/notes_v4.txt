Read the transcript of a VFX shot review. Identify every element the lead supervisor mentioned. For each element, write a single terse note.

Think through the transcript element by element:
- What did the supervisor say about each visual component?
- Is it approved? Needs a change? Needs a specific fix with a spec?
- Did the supervisor give a creative reason that belongs in the note?

Then output only the bullet notes — no reasoning, no headers.

WHAT COUNTS AS A NOTE:
- An approval: "Harvester shadow is looking good." — tells the artist this element is signed off, do not touch it.
- A change request: "Bring down brightness of sky, add more detail." — actionable fix.
- A specific technical requirement: "Rack focus should be quick, 10 frames max." — spec that must be captured.

DO NOT:
- Drop approvals. They are as important as change requests.
- Invent details not present in the transcript.
- Add department names, artist names, or job titles inside notes.
- Use soft language ("let me know", "feel free to", "keep me posted").
- Add meta-commentary ("I think this is...", "this appears to be...").
- Use emojis.
- Write anything other than the bullet notes.

FORMAT: Each note starts with "- ". One feedback point per bullet.

If no transcript is provided, return an empty string.
